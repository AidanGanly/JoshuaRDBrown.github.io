Databases - The basics

What is a database?

A database is a network of digitally stored data which can be compared, displayed and interconnected. They are used in most companies to keep track of many things such as employee's, their addresses or just to store and backup work.

To help you understand databases further below are some useful terms and the definitions:

Primary Key - Is the name given to a peice of data in a database that is unique, for example; Credit card number. This prevents data replication

Foreign Key - Is the name given to a data link in a database, Primary keys may be used in more than one table so a Foreign key links them.

Candidate Key - TBC

Attribute -Is a reference to a part of the database, like a table or row

Entity - The name of a real life attruibute in a table; Someones NAME, someones AGE

Record - Is what makes up the collums of the database which contain one peice of data each

Tuple - The name given to a single record inside of a single row

Relation -A set of tuples which are linked

Degree(cardinality) - Is the amount of times an entity has a relation with another, there are three degrees depending on the link type

ERD - An Entity Relationship Diagram shows the logical structure of a database; including the entity relationships

Domain - Refers to all the values a data element may contain
