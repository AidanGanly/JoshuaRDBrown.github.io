DBMS

What is a DBMS?

DBMS; which stands for DataBase Management System is a type of system software dedicated to creating and managing databases. This is an easy way to keep on top of data in a database. The main focus of a DMBS is to maintain a database which is easily accessable and which has a consistant organisation of data. It is also accountable for the security of a database, the integrity of the data and carrying out administrative prosedures.

API diagram of how a DBMS aranges data into a database.

A DBMS offers physical and logical data independence for a database.

Alternatives to DBMS:

There are many alternatives to the standard DMBS that you should be aware of:

In-memory database

Firstly there is in-memory databases. This database processes data using the system memory for faster results and lowered latency. It provides the option to optimize data management compared to normal disk-based data storing.

NoSQL

Another alternative is NoSQL databases. This is a specific system that does not follow the traditional database model, it has increased scalability and performance and is more capable for handling real time logical operations, such as those used for online banking transations for example.

Columnar databases

This database is a variation of a DMBS, however data is stored in columns instead of rows. The main purpose of this database is to increase read / write time from a hard disk to quickly return quries.

Advantages of DBMS:

There are many advantages of using a DBMS, some include:

  1. DBMS gives users and programmers access to use the same data while mantaining data integrity. This means data is better protected and organised instead of just iterations of the same data stored in new files.
  2. The DBMS provides a central hub for all data stored so that can be accessed by multiple users at once.
  3. DMBS also ensures data abstraction and independence inside of any given database. It also gives maximum security of all data stored and is less likely for a company to have to deal with a data breach.
  4. DMBS is also used for administrative purposes, all activity that goes on in the database is logged and saved to the admins computer (File deleted, file created) and by which user.
  5. DBMS can also recover very well from crashes and errors making recovery hassle free and reliablity high.

Disadvantages of DBMS:

  1. DMBS supports multiple functionality to give the user the best experience, the back end software has become complex. Only experienced users will get the most out of using one.
  2. Because of the point above, it uses large amount of memory. It also needs large memory and a high performance CPU that supports multicore processing to run
  3. As said in the advantges, DMBS uses a centralized system, this means all the users from all over the world access this database. Furthermore, any failures of the DBMS will impact all the users.
  4. DBMS is generalized software, it is written work on the entire systems rather than a specific one. This means some of the application will run slower than programmed.

Types of DBMS

SQLite

Firstly we have SQLite, which is a serverless database which uses the a C programming library. It embedds SQL into a console in the application so quries can be run and tables can be created directly. It is one of the most popular for it's easy to use interface and its link to many programming languages.

Microsoft access

Next there is Microsoft access, this is more for beginner programmers or small business' without any technical experiences. It has a export and import feature so data can be sent or taken from other Microsoft office applications. It's also easier than a client-side server and you can directly publish your database to the web.

Oracle

Oracle is a cloud based database server, this means migration from system to system is easy. This makes data more secure and less vunerable to being deleted or stolen. It also works well with Linux and windows platforms. It is also extremely hardware flexible, variations in database hardware still allow all features of Oracle to be used.
